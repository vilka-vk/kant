# Figma → CSS Design System Workflow

Верхнеуровневая инструкция по переносу дизайн-системы из Figma в статический HTML+CSS проект с использованием Figma MCP Server.

---

## Цель

Создать **CSS-файл с дизайн-токенами** (CSS Custom Properties), точно соответствующими переменным и стилям в Figma, и **файл компонентных стилей**, построенный исключительно на этих токенах. Такая структура обеспечивает:

- Единственный источник правды — Figma
- Быстрый перенос изменений: правка одного значения в `tokens.css` обновляет весь проект
- Прозрачную связь между Figma-именами и CSS-переменными

---

## Предварительные требования

| Инструмент | Зачем |
|---|---|
| **Figma MCP Server** | Программный доступ к дизайну (контекст, скриншоты, переменные, ассеты) |
| **Cursor / IDE с MCP** | Вызов MCP-инструментов прямо из среды разработки |
| **Google Fonts** | Подключение шрифтов, используемых в Figma (или локальные файлы шрифтов) |

---

## Шаг 0. Подготовка

1. Убедиться, что Figma MCP Server подключён и авторизован.
2. Получить ссылку на фрейм дизайн-системы в Figma (пример: `https://www.figma.com/design/:fileKey/:fileName?node-id=X-Y`).
3. Извлечь из URL:
   - **fileKey** — сегмент после `/design/`
   - **nodeId** — значение `node-id`, заменив `-` на `:` (например, `5-561` → `5:561`)
4. Создать файловую структуру проекта:

```
project/
├── assets/
│   ├── icons/          ← SVG-иконки из Figma
│   └── images/         ← Растровые изображения
├── css/
│   ├── tokens.css      ← Дизайн-токены (CSS Custom Properties)
│   └── components.css  ← Компонентные стили на основе токенов
└── index.html          ← Страница
```

---

## Шаг 1. Сбор данных из Figma

Выполнить **последовательно** три MCP-вызова:

### 1.1. `get_variable_defs`

```
get_variable_defs(fileKey, nodeId)
```

Возвращает **именованные переменные** (цвета, шрифты) в формате `"Figma/Path": value`. Это основа для CSS-токенов.

### 1.2. `get_design_context`

```
get_design_context(fileKey, nodeId, clientLanguages="html,css")
```

Возвращает:
- Код (React+Tailwind) — как **референс** структуры и значений
- Скриншот — визуальная цель
- Список `styles` — все именованные стили дизайн-системы
- URL ассетов (иконки, изображения)

### 1.3. `get_metadata`

```
get_metadata(fileKey, nodeId)
```

Возвращает XML-дерево с node ID, именами слоёв, позициями и размерами. Полезно для навигации по вложенным узлам.

### 1.4. `get_screenshot`

```
get_screenshot(fileKey, nodeId)
```

Скриншот всего фрейма — визуальный ориентир на протяжении всей работы.

## Шаг 2. Создание файла токенов (`tokens.css`)

### Принципы именования

| Figma-путь | CSS-переменная |
|---|---|
| `Accent/Accent 1` | `--color-accent-1` |
| `Text/Paragraph` | `--color-text-paragraph` |
| `Background/Background 1` | `--color-bg-primary` |
| `Dividers/Divider 1` | `--color-divider` |
| `Display` (font style) | `--text-display-size`, `--text-display-weight`, ... |

### Правила

1. **Каждая CSS-переменная сопровождается комментарием** с Figma-путём:
   ```css
   /* Figma: Accent/Accent 1 */
   --color-accent-1: #485C11;
   ```

2. **Категории токенов** внутри `:root`:
   - **Colors** — все цвета из `get_variable_defs`
   - **Fonts** — семейства шрифтов (`font-family`)
   - **Typography** — для каждого текстового стиля: `size`, `weight`, `line-height`, `letter-spacing`
   - **Spacing** — повторяющиеся отступы из дизайна
   - **Border Radius** — закругления
   - **Shadows** — тени
   - **Icon Size** — базовый размер иконок

3. **Конвертация letter-spacing**: Figma хранит его в **процентах** от `font-size`. Формула:
   ```
   letter-spacing (px) = font-size (px) × percentage / 100
   ```
   Например: `font-size: 60px`, `ls: -3%` → `letter-spacing: -1.8px`

4. **Шрифты**: добавить Google Fonts `<link>` или `@font-face` в HTML/CSS, точно совпадающие с `font-family` из Figma.

## Шаг 3. Создание файла компонентов (`components.css`)

### Правила

1. **Все значения — только через `var(--token)`**. Захардкоженных цветов, размеров, шрифтов быть не должно.

2. **Типографические классы** — по одному на каждый текстовый стиль Figma:
   ```css
   .text-display {
     font-family: var(--font-display);
     font-size: var(--text-display-size);
     font-weight: var(--text-display-weight);
     line-height: var(--text-display-line-height);
     letter-spacing: var(--text-display-letter-spacing);
     color: var(--color-text-headline);
   }
   ```

3. **Компонентные классы** (кнопки, навигация, карточки и т.д.):
   - Базовый класс (`.btn`) с общими стилями
   - Модификаторы (`.btn-primary`, `.btn-linkout`) для вариантов
   - Hover-состояния через `:hover` с переходами

4. **Каждый компонент аннотирован** Figma node ID в комментарии:
   ```css
   /* Figma: Button (node 5:611) */
   .btn { ... }
   ```

---

## Шаг 4. Скачивание ассетов

1. URL ассетов из `get_design_context` действительны **7 дней**.
2. Скачать все SVG-иконки, логотип и изображения **локально**:
   ```
   assets/icons/icon-name.svg
   assets/images/photo-1.jpg
   assets/logo.svg
   ```
3. В HTML использовать **локальные пути**, а не URL Figma MCP.

---

## Шаг 5. Сборка HTML-страницы

1. Подключить шрифты, `tokens.css`, `components.css`.
2. Разметить секции, соответствующие фрейму Figma.
3. На каждый HTML-элемент добавить `data-figma-node="X:Y"` — атрибут для трассировки до Figma-узла:
   ```html
   <p class="text-h1" data-figma-node="5:566">
     This Product feature is the most important
   </p>
   ```
4. Использовать исключительно классы из `components.css`.

---

## Шаг 6. Валидация

1. Открыть HTML-страницу в браузере.
2. Сравнить со скриншотом из Шага 1.4.
3. Проверить:
   - [ ] Типографика (шрифт, размер, вес, межстрочный, межбуквенный)
   - [ ] Цвета (фон, текст, акценты, разделители)
   - [ ] Иконки (размер, цвет)
   - [ ] Кнопки (padding, radius, hover-состояния)
   - [ ] Отступы и выравнивание
   - [ ] Навигация (десктоп, планшет, мобильная)

## Процесс обновления при изменениях в Figma

### Если изменились токены (цвета, шрифты, размеры):

1. Повторить `get_variable_defs` → получить обновлённые значения.
2. Найти соответствующую CSS-переменную по комментарию `/* Figma: ... */`.
3. Обновить значение в `tokens.css`.
4. Изменение автоматически каскадирует на все компоненты.

### Если изменилась структура компонента:

1. Повторить `get_design_context` для конкретного node ID.
2. Обновить соответствующий класс в `components.css`.
3. При необходимости обновить HTML-разметку.

### Если добавлены новые элементы:

1. Получить `get_design_context` для нового узла.
2. Добавить новые токены в `tokens.css` (если появились новые переменные).
3. Добавить новый компонентный класс в `components.css`.
4. Добавить разметку в HTML.

## Сводная таблица MCP-инструментов

| Инструмент | Что возвращает | Когда использовать |
|---|---|---|
| `get_variable_defs` | Именованные переменные (цвета, шрифты) | Первичный сбор токенов и при обновлениях |
| `get_design_context` | Код + скриншот + стили + ассеты | Основной инструмент для реализации |
| `get_metadata` | Структура узлов (XML) | Навигация по сложным фреймам |
| `get_screenshot` | Скриншот узла | Визуальная валидация |

---

## Ключевые принципы

1. **Figma — единственный источник правды** для визуальных решений.
2. **Токены — единственная точка связи** между Figma и CSS.
3. **Никаких хардкодов** в компонентных стилях — только `var(--token)`.
4. **`data-figma-node` атрибуты** обеспечивают трассируемость каждого элемента.
5. **Комментарии с Figma-путями** в `tokens.css` упрощают поиск при обновлениях.
6. **Ассеты — локально**, не зависеть от временных URL Figma MCP.
